01 Churn Rate Analysis

Objective:

This project aims to analyze customer churn for a telecommunications company in order to identify key patterns and enhance retention strategies, using Excel as the primary tool.

What was done:

The data underwent a thorough cleaning process, with filters applied to refine the analysis. Pivot tables were utilized to explore customer demographics, financial data, and overall churn metrics. Additionally, interactive dashboards were created in Excel to facilitate clearer insights into the findings.

Results:

The analysis revealed that 26.86% of customers have churned, leading to a loss of $1,367,515 in revenue. It was identified that the primary reason for customer attrition is the availability of better services from competitors. Notably, the segment with account lengths between 1 and 10 months displayed the highest churn concentration. Furthermore, nearly 90% of churned customers were found to have an international plan and a month-to-month contract.

02 Engineering Project Management

Objective:

The objective of this project was to extract earned value information from various projects through the integration of Oracle Primavera P6 with Power BI, thus enabling a more efficient and effective analysis of project performance.

What was done:

SQL queries were implemented for data modeling and to carry out the ETL (Extract, Transform, Load) process, enabling the collection, transformation, and loading of key project data. Using this information, an interactive dashboard was created in Power BI to clearly and effectively visualize project progress and performance, presenting key performance indicators and earned value curves.

Results:

The project resulted in a significant improvement in the visibility of project progress, allowing management teams to make more informed decisions. Additionally, automating the data extraction process eliminated the need for manual tasks, saving time and reducing errors in information management. The integration of Primavera P6 and Power BI proved to be a valuable tool for project planning and control across various engineering disciplines.

03 Supply Chain Optimization: Make vs Buy Analysis

Objective:

The goal of this project was to develop a "Make versus Buy" analysis tool using Power BI to optimize supply chain analytics. This approach highlighted Power BI’s advantages over traditional spreadsheet tools and enabled the development of a model that could be expanded to include additional cost considerations and supply chain aspects.

What was done:

During development, a quote analysis tool was built, and costs for different production volumes were calculated. Additionally, internal manufacturing cost data was integrated to provide a deeper analysis. The model was expanded to cover other key aspects of the supply chain, enhancing its usefulness in decision-making processes.

Results:

The project produced a robust "Make versus Buy" analysis tool that enables effective comparison between in-house manufacturing costs and external purchase costs. Power BI demonstrated clear advantages over traditional spreadsheets by facilitating dynamic visualization and data integration. The resulting model is adaptable and can be extended to cover more areas of the supply chain, becoming a valuable resource for the organization.
